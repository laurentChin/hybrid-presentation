Plusieurs approches

Web pur

  • Coquille générée par Cordova
  • Intelligence en JavaScript, structure en HTML, présentation en CSS
  • Pas d'intervention de développeur natif (hors plugin)
  • Doit être géré comme un projet web (coquilles non versionnées, lead web)

Une stack web

⇮ ⇮ ⇮

Dans quel cas ?

  • faible besoin de natif sur des fonctionnalités communes
  • animation non basées sur le défilement
  • scope de compatibilité favorable (Android >= 4.4, pas de Windows Phone)

Natif + Web

  • Coquille sur mesure pour chaque plateforme
  • responsabilités réparties
    • Présentation pour le web
    • Navigation pour le natif
    • Intelligence selon le cas
  • Communication web / natif à mettre en place
  • Doit être géré comme un projet natif (lead natif)

Dans quel cas ?

  • besoin de natif important et/ou spécifique
  • Beaucoup d'animations
  • scope de compatibilité favorable (Android >= 4.4, Windows Phone ?)

Les problèmes

La performance

JavaScript est monothreadé

Causes de surcharges

  • Animations modifiants la structure visuelle (hors 3d)
  • callback d'évènements fréquents (touchmove, "scroll")
  • Non utilisation du prototypage
  • Accès aux propriétés déclenchants un reflow
  • data-binding non natif (web)

La compatibilité

  • 4 versions d'android au dessus de 10%
  • Android <4.4 = IE8 (30% de parts de marché)
  • Windows Phone peu supporté

Une solution possible

Remplace la webview native par un chromium récent
